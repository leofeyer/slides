Contao Manager

Verwaltung der Contao-Installation im Browser

Beginn der Public Beta-Phase mit dem Release von Contao 4.4 LTS

Installation

  • Erstellen des Ordners /web im Zielverzeichnis der Webseite
  • Routing der Domain auf den Ordner /web
  • Download des Contao Manager als contao-manager.phar
  • Übertragung der .phar-Datei per FTP im Binärmodus*
  • Umbennenen der Datei auf dem Server in contao-manager.phar.php
  • Aufruf der Datei contao-manager.phar.php im Browser

*Die Datei kann auch direkt auf den Server geladen werden, zum Beispiel mittels wget oder curl.

Migration

Die Migration von Contao 3 auf Contao 4 muss manuell erfolgen.

Vorgehen

  • Installation von Contao 4 parallel zu Contao 3
  • Installation aller benötigten Erweiterungen
  • Kopieren der Ordner files und templates
  • Kopieren der Contao 3-Datenbank
  • Aufruf des Installtools der Contao 4-Installation
  • Manuelles Nacharbeiten (z.B. wegen geänderter CSS-Klassen)

Formular-Markup

Formularfelder haben nun systemweit ein einheitliches Markup:

<div class="widget widget-text">
  <label for="ctrl_16">Name</label>
  <input type="text" name="name" id="ctrl_16" class="text">
</div>

Formulare werden mit dem button-Element abgeschickt:

<div class="widget widget-submit">
  <button type="submit" id="ctrl_17" class="submit">Abschicken</button>
</div>

Edge Side Includes

Inserttags werden als ESI-Tags gerendert und vom HTTP-Cache aufgelöst.

Vorteile

  • Dynamische Fragmente auf gecachten Seiten möglich
  • Entfall von Vary: User-Agent für die Body-Klasse
  • Erhebliche Reduzierung der Dateien des HTTP-Caches
